# Tribe Block University Course Contribution Workflow

GitHub is the primary source for course content. The platform stores course metadata in the database, while lesson source files live in the Tribe Block University GitHub organization.

Repository: https://github.com/Tribe-Block-University

Foundation reference source: https://developer.mozilla.org/en-US/

## Review Flow

1. A contributor edits an existing course or submits a new course through GitHub.
2. Admins and mentor reviewers review the submission.
3. Approved changes are merged into the course repository.
4. Merged content is synced into the platform as modules, lessons, quizzes, IDE exercises, and projects.
5. Approved contributors become eligible for the Teaching Commission Program.

## Access Model

Free foundational programming content should include lessons, modules, IDE practicals, quizzes, exams, completion progress, and fundamental course badges.

Plus and Pro learners can access guided real-world projects with IDE checkpoints and structured build paths.

Pro learners can request NFT certificates and advanced credential proof.

## Reward Categories

Final G$ reward amounts are determined by the Tribe Block administration team.

- New course
- New lesson
- Major course expansion
- Lesson improvement
- Documentation improvement
- Example enhancement
- Bug fix
- Platform improvement
- IDE improvement
- Content review
- Course validation
- Translation contribution
